## 📸 Добавление изображений профессий

### Вариант 1: Автоматическое копирование (рекомендуется)

1. Поместите все 6 PNG-файлов в одну папку
2. Переименуйте их согласно таблице ниже
3. Запустите скрипт:

```bash
./copy-images.sh /путь/к/папке/с/изображениями
```

### Вариант 2: Ручное копирование

Скопируйте изображения в `public/images/` с именами:

| Файл | Описание изображения |
|------|---------------------|
| `waitress.png` | Девушка в чёрной одежде с чашкой кофе |
| `manicure.png` | Мастер маникюра с красными волосами |
| `blogger.png` | Девушка в белом халате снимает контент |
| `gas-station.png` | Неоновая вывеска заправки ночью |
| `courier.png` | Человек в маске с наушниками и рюкзаком |
| `anyone.png` | Две девушки с покупками на эскалаторе |

### Вариант 3: Быстрое копирование из терминала

```bash
# Если изображения в Downloads:
cp ~/Downloads/image1.png public/images/waitress.png
cp ~/Downloads/image2.png public/images/manicure.png
cp ~/Downloads/image3.png public/images/blogger.png
cp ~/Downloads/image4.png public/images/gas-station.png
cp ~/Downloads/image5.png public/images/courier.png
cp ~/Downloads/image6.png public/images/anyone.png
```

## 🔄 После добавления изображений

1. Обновите страницу в браузере (⌘+R или Ctrl+R)
2. Прокрутите до секции "Подойдёт всем, кого принято благодарить"
3. Убедитесь, что карточки отображают реальные фотографии

## 📝 Примечание

Если изображения не добавлены, карточки будут показывать градиентный фон как fallback. Это нормально и не влияет на работу сайта.

## ❓ Проблемы?

- **Изображения не отображаются?** Проверьте имена файлов и расширение (.png)
- **Сервер не запускается?** Выполните `npm install` и `npm run dev`
- **Ошибки в консоли?** Откройте DevTools (F12) и проверьте вкладку Console
