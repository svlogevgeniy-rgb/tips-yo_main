# 📱 Документация User Flow - Сервис чаевых tipsyo

## 🎯 Обзор реализованного flow

Реализован полный пользовательский сценарий для веб- и мобильной версии сервиса чаевых с адаптивным дизайном и роутингом.

---

## 🗺️ Структура роутов

### 1. Главная страница (Landing)
**URL:** `/`
**Компонент:** `HomePage`
**Описание:** Стартовая страница с информацией о сервисе

**Элементы:**
- Header с навигацией и кнопками CTA
- Hero-секция с главным призывом
- Секции: "Для кого", "Как работает", "Тарифы"
- Footer

**Кнопки "Поблагодарить":**
- В header (desktop)
- В hero-секции
- В секции тарифов
- В мобильном меню

**Действие:** При клике → переход на `/employees`

---

### 2. Страница выбора сотрудника
**URL:** `/employees`
**Компонент:** `EmployeeSelection`
**Описание:** Список всех доступных сотрудников для благодарности

**Элементы:**
- Header с кнопкой "Назад" и заголовком
- Описание: "Выберите сотрудника, которого хотите поблагодарить"
- Список карточек сотрудников (EmployeeCard)
- Empty state (если сотрудников нет)

**Карточка сотрудника содержит:**
- Аватар (круглое фото)
- Имя сотрудника
- Должность/роль
- Стрелка вправо (desktop)

**Действие:** При клике на карточку → переход на `/employee/:id`

---

### 3. Страница оплаты чаевых
**URL:** `/employee/:id`
**Компонент:** `PaymentPage`
**Описание:** Страница оплаты чаевых для конкретного сотрудника

**Элементы:**
- Header с кнопками "Назад" и "Поделиться"
- Карточка с информацией о сотруднике:
  - Аватар (большой, круглый)
  - Имя
  - Должность
  - QR-код (сейчас placeholder)
  - ID сотрудника
  - Пояснение "Отсканируйте QR-код"
- Кнопки действий:
  - "Оплатить чаевые" (primary)
  - "Выбрать другого сотрудника" (outline)
- Информация о безопасности и комиссии

**Обработка ошибок:**
- Если сотрудник не найден → показывается карточка с ошибкой и кнопкой возврата

**Действие:** 
- "Назад" → возврат на `/employees`
- "Выбрать другого" → переход на `/employees`

## 🔄 Переходы между страницами

### 1. Главная → Выбор сотрудника

**Триггеры:**
- Клик на кнопку "Поблагодарить" в header
- Клик на кнопку "Поблагодарить" в hero-секции
- Клик на кнопку "Поблагодарить" в секции тарифов
- Клик на кнопку "Поблагодарить" в мобильном меню

**Реализация:**
```typescript
const navigate = useNavigate()
const handleThankClick = () => {
  navigate('/employees')
}
```

**Поведение:**
- Мгновенный переход без перезагрузки страницы (SPA)
- URL меняется на `/employees`
- Загружается компонент `EmployeeSelection`

---

### 2. Выбор сотрудника → Персональный QR-код

**Триггер:**
- Клик на любую карточку сотрудника

**Реализация:**
```typescript
const handleEmployeeClick = (employeeId: string) => {
  navigate(`/employee/${employeeId}`)
}
```

**Поведение:**
- Переход на `/employee/1` (где 1 - ID сотрудника)
- Загружается компонент `EmployeeQRPage`
- Компонент извлекает ID из URL параметров
- Находит сотрудника в данных по ID
- Отображает персональную информацию

---

### 3. Возврат назад

**Триггеры:**
- Кнопка "Назад" (стрелка) в header
- Кнопка "Выбрать другого сотрудника"

**Реализация:**
```typescript
navigate('/employees')  // или navigate('/')
```

## 📱 Адаптивный дизайн (Desktop vs Mobile)

### Подход: Single Route + Responsive CSS

**Принцип:** Один и тот же URL для desktop и mobile, различия реализованы через:
- Tailwind CSS breakpoints (`md:`, `lg:`)
- Flexbox/Grid адаптивные раскладки
- Условный рендеринг элементов

---

### Главная страница (HomePage)

#### Desktop (≥768px):
- Горизонтальная навигация в header
- Кнопки CTA в header
- Широкая компоновка секций
- Карточки профессий: 6 в ряд
- Двухколоночная сетка в "Как работает"

#### Mobile (<768px):
- Бургер-меню вместо навигации
- Скрытые кнопки в header (доступны в меню)
- Вертикальная компоновка
- Карточки профессий: 2 в ряд
- Одноколоночная сетка

**Реализация:**
```tsx
{/* Desktop Navigation */}
<nav className="hidden md:flex items-center space-x-8">
  {/* ... */}
</nav>

{/* Mobile Menu Button */}
<button className="md:hidden p-2">
  <Menu />
</button>
```

---

### Страница выбора сотрудника (EmployeeSelection)

#### Desktop:
- Карточки с горизонтальной раскладкой
- Аватар слева, информация по центру, стрелка справа
- Hover-эффекты
- Больший размер аватара (24x24)

#### Mobile:
- Карточки с вертикальной раскладкой
- Аватар сверху, информация снизу по центру
- Без стрелки
- Меньший размер аватара (20x20)
- Крупные кликабельные области

**Реализация:**
```tsx
<div className="flex flex-col md:flex-row items-center gap-4">
  <div className="w-20 h-20 md:w-24 md:h-24 rounded-full">
    {/* Avatar */}
  </div>
  <div className="text-center md:text-left">
    {/* Info */}
  </div>
  <div className="hidden md:block">
    {/* Arrow */}
  </div>
</div>
```

---

### Персональная страница QR (EmployeeQRPage)

- Широкая карточка с большими отступами (p-8)
- Крупный аватар (32x32)

- Компактная карточка (p-6)

## 🎨 Состояния и обработка ошибок

### Loading State
**Текущая реализация:** Данные загружаются синхронно (mock)
**Будущее:** Добавить состояние загрузки при API-запросах

```typescript
// Пример для будущей реализации
const [loading, setLoading] = useState(true)
const [employees, setEmployees] = useState<Employee[]>([])

useEffect(() => {
  fetchEmployees().then(data => {
    setEmployees(data)
    setLoading(false)
  })
}, [])
```

### Error State - Сотрудник не найден

**Реализация в EmployeeQRPage:**
```typescript
const employee = employees.find((emp) => emp.id === id)

if (!employee) {
  return (
    <div className="min-h-screen bg-gray-50 flex items-center justify-center">
      <Card className="p-8 text-center">
        <h2>Сотрудник не найден</h2>
        <Button onClick={() => navigate('/employees')}>
          Вернуться к выбору
        </Button>
      </Card>
    </div>
  )
}
```

### Empty State - Нет сотрудников

**Реализация в EmployeeSelection:**
```typescript
{employees.length === 0 && (
  <div className="text-center py-12">
    <p className="text-gray-500">Сотрудники не найдены</p>
  </div>
)}
```

## 🔧 Технические детали

### Роутинг
**Библиотека:** React Router v6
**Тип:** Client-side routing (SPA)

**Преимущества:**
- Мгновенные переходы без перезагрузки
- Поддержка истории браузера (back/forward)
- Динамические параметры в URL
- SEO-friendly URLs

### Навигация
**Хук:** `useNavigate()`
**Параметры:** `useParams<{ id: string }>()`

### Адаптивность
**Фреймворк:** Tailwind CSS
**Breakpoints:**
- `sm:` 640px
- `md:` 768px
- `lg:` 1024px
- `xl:` 1280px

## 📝 Допущения и решения

### Допущение 1: Структура данных
Создана простая структура Employee с базовыми полями. В реальном приложении может потребоваться:
- Дополнительные поля (email, phone, rating)
- Связь с организацией/заведением
- История транзакций

### Допущение 2: QR-код
Используется placeholder вместо реального QR-кода. Для продакшена нужно:
- Библиотека для генерации QR (например, `qrcode.react`)
- Уникальный URL для каждого сотрудника
- Возможность скачать/поделиться QR

### Допущение 3: Аутентификация
Текущая версия не требует авторизации. Для полноценного сервиса нужно:
- Регистрация/вход для сотрудников
- Защищённые роуты
- Управление профилем

### Допущение 4: Оплата
Кнопка "Оплатить чаевые" пока не функциональна. Требуется:
- Интеграция платёжной системы
- Форма ввода суммы
- Подтверждение платежа
