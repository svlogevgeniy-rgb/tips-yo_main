# Tips'yo Landing & Documentation

Одностраничный лендинг сервиса бескэшевых чаевых Tips'yo (продуктовое имя Tips’yo) и основной источник документации, перенесённой из `Tips_yo_dev`.

## Что внутри

- **Продакшен-лендинг:** React 18 + TypeScript + Vite + Tailwind + shadcn/ui.
- **Документация:** `docs/product`, `docs/engineering`, `docs/ui`, `docs/setup`, `docs/operations`, `docs/changelog`.
- **Утилиты для ассетов:** `copy-images.sh`, `check-images.sh`.

## Быстрый старт

```bash
npm install
npm run dev
```

Локальная разработка доступна на [http://localhost:5173](http://localhost:5173).

Production-сборка:

```bash
npm run build
npm run preview   # smoke-тест собранной версии
```

Подробности по окружению и настройке см. в `docs/setup/local.md` и `docs/setup/env.md`.

## Видение и ценность

- **Миссия:** Tips’yo создаёт на Бали стандарт цифровых чаевых и сервисов «за столом», делая чаевые удобными гостям, прозрачными персоналу и управляемыми для владельцев заведений.
- **Целевая аудитория:** первично рестораны, кафе, бары, кофейни и другие F&B площадки на Бали; вторично — их сотрудники (официанты, бармены, бариста, хостес) и гости (туристы и экспаты).
- **Ценностное предложение:** гостю — быстрый cashless сценарий с понятным адресатом чаевых, персоналу — честное распределение и прогнозируемый доход, заведению — контроль и аналитика без смены текущей платёжной инфраструктуры (Midtrans, QRIS, e-wallet, карты).
- **Стратегические принципы:** Bali-first и people-first подход, мобильные интерфейсы по умолчанию, деньги всегда принадлежат заведению, архитектура готова к масштабированию на другие рынки ЮВА.

Полную версию продуктового видения см. в `docs/product/vision.md`.

## Структура лендинга

- **Header** — липкая шапка с логотипом, навигацией и CTA.
- **Hero** — главный экран с призывом и безопасностью платежей.
- **Quick Links** — быстрые ссылки-анкоры.
- **For Whom** — карточки профессий с реальными изображениями или fallback-градиентами.
- **How It Works** — сценарий QR/код.
- **Pricing** — тарифы и CTA.
- **Footer** — юридические ссылки, язык, копирайт.

## Работа с изображениями

1. Подготовьте PNG-файлы и поместите их в `public/images/` с именами `waitress.png`, `manicure.png`, `blogger.png`, `gas-station.png`, `courier.png`, `anyone.png`.
2. Для автоматизации используйте `./copy-images.sh <source-dir>`.
3. При отсутствии файлов карточки используют градиенты, так что сборка не ломается.

Подробности и примеры — `QUICK_START.md`.
