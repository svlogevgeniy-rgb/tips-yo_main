# UI Overview — Tips'yo

## Цели и принципы
- **Быстрота**: гость проходит сценарий чаевых за ≤3 взаимодействия.
- **Прозрачность**: заведение и персонал мгновенно видят статусы и сплиты.
- **Надёжность**: визуальный язык повторяет статусы Xendit, ошибочные состояния заметны.
- **Consistency**: все экраны используют единый UI Kit (tokens → atoms → molecules → widgets).

## Пользователи и сценарии
| Роль | Основные задачи |
|------|-----------------|
| Guest | Сканировать QR → выбрать сумму → оплатить через Payment Link/QRIS → увидеть «Спасибо». |
| Merchant Admin | Онбординг заведения, настройка сотрудников и split rule, контроль операций, экспорт, отслеживание вебхуков. |
| Staff | Видеть персональные суммы чаевых, графики за смену/неделю, статусы выплат. |
| Platform Admin | Мониторинг интеграций (ретраи, расхождения), ручные действия, аналитика пилота. |

## Ключевые экраны (Figma)
1. **Tip Landing** — node `47:1877`. Блоки: хедер заведения, Amount Pills, кастомная сумма, код сотрудника, CTA «Оплатить», ссылки на Terms/Support.
2. **Thank you / Receipt** — подтверждение оплаты, рекомендации, возможность поделиться QR, CTA «Вернуться в меню».
3. **Merchant Dashboard** — KPI-плитки, графики, список сотрудников, операции, настройки split, фильтры по периоду.
4. **Staff Dashboard** — карточка прогресса, цели на смену, история выплат, CTA «Обновить данные».
5. **Onboarding Wizard** — пошаговые формы KYC (данные заведения, банковские реквизиты, документы).
6. **Split Settings** — таблица сотрудников, проценты, предпросмотр распределений, состояния ошибок.
7. **Webhook Monitor** — лог событий (`eventId`, статус, retriable), фильтры, кнопка Retry/Details.

## Основные блоки
- **Layout**: Navbar + адаптивная панель (sidebar для desktop, sheet для mobile).
- **Action bar**: фиксированная зона для CTA/фильтров.
- **Cards**: KPI, операции, сотрудники.
- **Tables & Lists**: операции, сотрудники, webhooks.
- **Forms**: wizard, настройки split, TipForm.

## Ключевые действия
- Выбор суммы (preset/custom), ввод кода сотрудника, запуск оплаты.
- Создание/редактирование split rule, экспорт CSV.
- Просмотр/фильтрация операций, ретрай вебхуков.
- Отслеживание прогресса персонала (gamification badges).
