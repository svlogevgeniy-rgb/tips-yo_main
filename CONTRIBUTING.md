# Contributing Guide

## 1. Как работать с репозиторием

- Основная ветка — `main`. Для задач создавайте ветки `feature/<topic>`, `fix/<topic>`, `docs/<topic>`.
- Перед началом убедитесь, что локальная ветка синхронизирована: `git pull origin main`.

## 2. Коммиты

- Используйте формат `type: summary`, где `type ∈ {feat, fix, docs, chore}`.
- Один коммит — одна логическая причина изменения.

## 3. Перед PR

```bash
npm install        # при необходимости
npm run build
npm run preview    # быстрый smoke-тест
```

Проверьте, что lint/tests не падают (если в проекте появятся соответствующие команды).

## 4. Документация

- Продуктовые изменения — обновляйте `docs/product`.
- UI/UX правила — `docs/ui`.
- Настройки окружения и деплоя — `docs/setup`, `docs/operations`.
- Фиксируйте релевантные изменения в `docs/changelog.md`.

## 5. Code style

- Следуйте `docs/engineering/coding-conventions.md`.
- Используйте существующие UI-компоненты (`src/components/ui`) прежде чем создавать новые.
- Для работы с изображениями придерживайтесь правил из `docs/setup/local.md`.

## 6. Code review чек-лист

- ✅ UI соответствует дизайну, нет регрессий в адаптивности.
- ✅ CTA ведут на правильные ссылки.
- ✅ Нет хардкода секретов и чувствительных данных.
- ✅ Документация/README обновлены, если поведение поменялось.
